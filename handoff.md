## 当前最重要的已落地规则

### 1. 跨材料停留天数一致性

这条是目前最关键的确定性校验，已成功落地。

已能从以下材料提取天数并对比：

- 呈报表：`停留时间 XX 天`
- 邀请函翻译件：`在国外停留时间 XX 天`
- 预算审批意见表：`出访时间（天数）XX 天`

若不一致，程序直接报严重问题。

预算审批意见表天数现直接使用 `BudgetFacts.duration_days` 参与比对，避免预算事实层已识别天数、但一致性规则因 OCR 文本格式差异漏报。

### 2. 禁用词

- `参观 / 考察 / 调研`：按严重问题处理
- `学习`：
  - 在 `培训 / 访学 / 学术交流` 场景放过
  - 其他场景可作为禁用词
- 邀请函译文中的禁用词不判断
- 同类禁用词只报一条

### 3. 跨材料人员姓名一致性

- 从呈报表中高置信抽取“某某拟于……赴……”这类明确出访人员姓名
- 从团组人员名单中抽取“姓名 + 性别”名单行
- 呈报表明确出访人员未出现在人员名单时，程序直接报严重问题
- 不扫描邀请函和日程中的联系人姓名，避免把邀请人、外方联系人误判为团员

### 4. 跨材料人员出生日期一致性

- 从团组人员名单身份证号码中解析出生日期，不直接比较身份证号和护照号
- 从邀请函译文的受邀人员名单中按姓名抽取出生日期
- 仅对两份材料中同名人员的出生日期进行比较
- 出生日期不一致时程序直接报严重问题，并定位人员名单页和邀请函页
- 不把邀请函落款日期、访问起止日期等其他日期误当出生日期

### 5. 企业团组口径

团组类型必须以“组团单位”字段值为准，不再全文扫描 `公司` 等关键词。行程中的“航空公司”等文本不能作为企业团组依据。

组团单位字段值出现 `集团 / 公司 / 有限公司 / 股份公司` 等时，优先按企业团组处理。

组团单位字段值出现 `大学 / 学院 / 学校 / 研究院 / 研究所 / 实验室` 等时，优先按高校科研院所团组处理。

企业团组不直接因缺少以下内容判错：

- 列入计划情况
- 周末公务情况
- 翻译情况
- 是否学术交流团

但仍应关注公示情况。

### 6. 材料顺序与访学附件范围

- 常见材料顺序默认按：
  - `呈报表`
  - `人员名单`
  - `日程安排`
  - `预算表`
  - `情况说明`
- 对访学团组后附材料：
  - 默认只识别附件第一页
  - 不做附件通篇逐页扫描
  - 不拿附件正文做禁用词和细碎语义判断
